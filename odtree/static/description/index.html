Custom Tree Structure In ListView or KanbanView

Product Category Tree.

Employee Department Tree.

Usage

Extending a listview or a kanbanview, and specify the value of new attributes :
categ_property,categ_model,categ_parent_key(optional) ;

An typical inherited view declaration looks like this:

<record id="view_employee_tree_with_dpttree" model="ir.ui.view">
<field name="name">hr.employee.tree</field>
<field name="model">hr.employee</field>
<field name="inherit_id" ref="hr.view_employee_tree" />
<field name="arch" type="xml">
    <xpath expr="//tree" position="attributes">
        <attribute name="categ_property">department_id</attribute>  <----- the field relate to the tree Structure in the model
        <attribute name="categ_model">hr.department</attribute>     <----- the model of the tree Structure
        <attribute name="categ_parent_key">parent_id</attribute>    <----- (optional)if only want to get a single layer tree,do not specify a value for this attribute
    </xpath>
</field>
</record>
